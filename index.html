    Airstrikes 1.1.1 (beta) released

  • This update is simply just a hotfix. It fixes some minor bugs, such as a typo in the permissions ("Airstrikes.whatever" instead of "Airstrike.whatever"). It also adds the ability for admins to use as many airstrikes as they want (basically a bypass). If you don't happen to use the /admin command, the "Airstrikes.admin" permission has been added to accomplish the same.

    The update is now live!

    Airstrikes 1.1.2 (beta) released

  • This is a minor update but it is still highly recommended that you download this for stability reasons.

    This update has fixed some issues including: lag when using a ground effect, and /strike command not dealing enough damage. Auto airstrike has been tweaked to start after all plugins have finished loading, so that everything is nice and organized! Last, but certainly not least for this update is optimization! This is the most stable release of Airstrikes yet, so if you were experiencing any lag or abnormal frame-loss, this is the update for you!

    Airstrikes 1.2.0 released

  • The next major release of the Airstrike plugin is here!

    1.2.0 is officially out of beta stage meaning almost everything that was planned has been added, and most (if not all) bugs and glitches have been resolved. This update includes these notable changes...

    • - Added damage intensity option to config
    • - Added explosion effect ID option to config
    • - Added delay & duration time-frame options to config
    • - Added a maximum distance option (for setting max distance of manual airstrikes) to config
    • - Global messages now show who called an airstrike
    • - Airstrike coords no longer show in global chat (by default)
    • - Fixed issues with 'Shuffle' not properly randomizing Locations after each loop
